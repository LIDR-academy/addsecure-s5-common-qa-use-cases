Requirements Change Report

Change: IEEE 754 precision from 16 decimal places to 12 decimal places

Context: This repository implements a numeric equivalence validator between Python and C++ for the formula value = (a + b) * c. The normalization contract currently mandates exactly 16 decimal places with ROUND_HALF_EVEN. The proposed change reduces precision to 12 decimal places.

Scope: The change affects the core normalization logic, its documentation, all unit/BDD/mutation tests that assert on the exact output format, and every hardcoded expected value string containing 16 decimal digits.

Key risk: Several rounding boundary tests (e.g. 5e-17, 2.5e-16) were chosen specifically because they sit at the half-way point of 1e-16. At 12 decimal places the half-way points shift to 1e-12, so those test inputs must be redesigned.
